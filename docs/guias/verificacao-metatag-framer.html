Verificar o domínio no Meta pelo Framer

Passo a passo para o time verificar o subdomínio da página de captação no Gerenciador de Negócios do Meta, usando o método de metatag — o único que funciona com páginas hospedadas no Framer.

lp.lucrocomia.com.br Portfólio · ANEX BM Funil · QUIZZ
A metatag

O código que vai para o site

Este é o código de verificação. Ele é o mesmo para todo o portfólio ANEX BM, então não há risco de trocar o código — é só copiar e colar exatamente como está.

Metatag de verificação
<meta name="facebook-domain-verification" content="ezf5ecxc7l2577e2zd4pcve3pedxx6" />
Passo a passo

Cinco passos, cerca de 5 minutos

  1. Abra o site no Framer
    Entre no projeto do lp.lucrocomia.com.br dentro da conta Framer.
  2. Vá em Site Settings → General → Custom Code
    Clique na engrenagem de configurações do site e desça até a seção Custom Code.
  3. Cole a metatag no campo “Start of <head> tag”
    Tem que ser esse campo, o início do <head>. Não cole no corpo da página nem em bloco de código dentro do layout: se ficar fora do <head>, o Meta não reconhece.
  4. Salve e clique em Publish (republicar)
    O passo mais esquecido. No Framer a metatag só vai para o ar depois de republicar o site. Sem republicar, nada muda na página.
  5. Volte ao Meta e clique em “Verificar domínio”
    Gerenciador de Negócios → Configurações → Domínios → selecione lp.lucrocomia.com.br → botão Verificar domínio. O status deve virar Verified.
Confira antes de clicar em verificar. Depois de republicar no Framer, abra lp.lucrocomia.com.br, aperte Ctrl+U (exibir código-fonte) e procure por facebook-domain-verification. Se o código aparecer ali dentro, a verificação vai passar.
Diagnóstico

Por que não verificava antes

Não era erro de quem tentou. O subdomínio tem uma particularidade técnica que trava os outros métodos:

  • O domínio raiz estava verificado, mas o subdomínio não. O Meta trata lucrocomia.com.br e lp.lucrocomia.com.br como coisas separadas. Verificar a raiz não verifica o lp. automaticamente.
  • O método por DNS (registro TXT) é impossível neste caso. O lp. é um CNAME apontando para o Framer, e pela regra do DNS um host que é CNAME não aceita registro TXT junto. Por isso qualquer tentativa pelo DNS falhava, por mais que estivesse tudo certo.
  • A metatag ainda não estava publicada na página. O código não constava no <head> do site — provavelmente por não ter republicado no Framer depois de colar.
Ou seja: a metatag é o caminho certo para páginas em Framer, não um plano B. O código é o mesmo que já verificou a raiz com sucesso; muda só o lugar onde ele fica — dentro do site, no <head>, em vez do DNS.
