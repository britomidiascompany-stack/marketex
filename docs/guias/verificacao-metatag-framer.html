Como verificar o domínio no Meta pelo Framer

Guia rápido para o time · Funil QUIZZ (ANEX BM)
Domínio: lp.lucrocomia.com.br  ·  Portfólio: ANEX BM  ·  Método: metatag no Framer

Este guia mostra o passo a passo para verificar o subdomínio lp.lucrocomia.com.br no Gerenciador de Negócios do Meta. Ele usa o método de metatag, que é o único que funciona quando a página está hospedada no Framer. Siga na ordem — leva uns 5 minutos.

🏷️ A metatag

Este é o código que precisa ir para dentro do site. Ele é o mesmo para todo o portfólio ANEX BM, então não tem risco de errar o código:

METATAG DE VERIFICAÇÃO <meta name="facebook-domain-verification" content="ezf5ecxc7l2577e2zd4pcve3pedxx6" />

📋 Passo a passo

  1. Abra o site no Framer
    Entre no projeto do lp.lucrocomia.com.br dentro da sua conta Framer.
  2. Vá em Site Settings → General → Custom Code
    Clique na engrenagem de configurações do site e desça até a seção Custom Code.
  3. Cole a metatag no campo "Start of <head> tag"
    Tem que ser esse campo (início do <head>). Não cole no corpo da página nem em bloco de código dentro do layout — se ficar fora do <head>, o Meta não reconhece.
  4. Salve e clique em Publish (republicar)
    Passo mais esquecido. No Framer a metatag só vai para o ar depois de republicar o site. Sem republicar, nada acontece.
  5. Volte ao Meta e clique em "Verificar domínio"
    Gerenciador de Negócios → Configurações → Domínios → selecione lp.lucrocomia.com.br → botão Verificar domínio. O status deve virar Verified.
✅ Dica antes de clicar em verificar: depois de republicar no Framer, dá pra conferir se a tag entrou abrindo lp.lucrocomia.com.br, clicando com o botão direito → "Exibir código-fonte da página" (Ctrl+U) e procurando por facebook-domain-verification. Se aparecer o código lá dentro, a verificação vai passar.

🤔 Por que não estava verificando antes

Não era erro de quem tentou. O subdomínio tem uma particularidade técnica que trava os outros métodos:

  • O domínio raiz estava verificado, mas o subdomínio não. O Meta trata lucrocomia.com.br e lp.lucrocomia.com.br como coisas separadas. Verificar a raiz não verifica o lp. automaticamente.
  • O método de DNS (registro TXT) é impossível neste caso. O lp. é um CNAME apontando para o Framer, e pela regra do DNS um host que é CNAME não aceita registro TXT junto. Por isso qualquer tentativa pelo DNS falhava, por mais que estivesse tudo certo.
  • A metatag ainda não estava publicada na página. O código não constava no <head> do site — provavelmente por não ter republicado no Framer depois de colar.
Ou seja: a metatag é o caminho certo para páginas em Framer, não um plano B. O código é o mesmo que já verificou a raiz com sucesso; muda só o lugar onde ele fica (dentro do site, no <head>, em vez do DNS).
